Cross-root inner surface — label and help from Light DOM

Based on the ariaDescribedByElements CodePen POC. Each component renders its interactive ARIA surface inside shadow (native <input> or role="progressbar"). Label and help stay in Light DOM and are linked with inner.ariaLabelledByElements and inner.ariaDescribedByElements — shadow → light only; no ElementInternals, no string ID attributes.

                        Works: shadow inner surface → light DOM label/help (same direction as
                        inner button → tooltip in the CodePen).

                        Does not work: host or light DOM → shadow-resident label/help; string
                        aria-labelledby / aria-describedby from shadow cannot cross

  •

                        external or slotted light labels — aligns with 2nd-gen default role placement.

  • 
                        Inspect: DevTools → Accessibility → expand custom element → shadow root
