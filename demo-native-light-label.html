Shadow role — light DOM label and help

The ARIA role lives on an inner shadow DOM element, not the host. For textfield and checkbox that element is a native <input>; for progressbar and combobox it is a <div role="...">. Label and help are light DOM siblings to the component. They are wired by setting ariaLabelledByElements and ariaDescribedByElements directly on the shadow element — source is in shadow DOM, target is in the lighter parent DOM, so the element reference API accepts the assignment.

Note: <label for="id"> cannot target an input inside a shadow root because shadow IDs are not globally accessible. The element reference API is the correct wiring mechanism even for native inputs.
