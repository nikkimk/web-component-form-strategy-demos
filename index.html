Form fields strategy demos

Interactive PoCs exploring how label and description connect across Shadow DOM boundaries for textfield, checkbox, progress bar, and combobox components. In both demos the ARIA role lives on an inner shadow DOM element, not the custom element host.

Project README · Open in StackBlitz

Shadow DOM label and description

Label and description live inside the shadow DOM with <slot name="label"> and <slot name="description"> for consumer-provided text. The role element is wired via aria-labelledby and aria-describedby using same-root IDs — no cross-root ID references needed. The combobox uses Element.ariaActiveDescendantElement for its active option.

  • Shadow role — shadow DOM label and description (all four controls)

Light DOM label and description siblings

Label and description are plain light DOM siblings of the custom element — no slots, no shadow markup for them. Each component reads its host’s aria-labelledby and aria-describedby attributes, resolves the referenced elements in the document, and wires them to the inner role element using Element.ariaLabelledByElements and Element.ariaDescribedByElements. These element reference APIs work cross-root (shadow source → light DOM target).

  • Shadow role — light DOM label and description siblings (all four controls)
